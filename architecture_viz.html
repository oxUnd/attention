1全局鸟瞰

本项目在约 2000 行 C 代码中实现了完整的 Encoder-Decoder Transformer，外加分词器、训练循环和 KV Cache 推理。四大模块各司其职：

整体数据流一句话版

文本 → Tokenizer → token IDs → Embedding查表 → + PE → Encoder → Decoder → Logit投影 → Softmax → 采样 → 文本

如果错了就反向： Loss → ∂L/∂logits → Backward through Decoder → Backward through Encoder → ∂L/∂Embedding(散点加法) → Adam更新权重

2数据结构

Tensor3D — 三维张量

typedef struct {
    int batch_size;  // 批次
    int seq_len;     // 序列长度
    int d_model;     // 隐藏维度
    float *data;     // 行优先连续内存
} Tensor3D;

想象一本笔记本：batch_size 本，每本 seq_len 页，每页 d_model 个格子。

Matrix — 权重矩阵

typedef struct {
    int rows;    // 输出维度
    int cols;    // 输入维度
    float *data; // 行优先
} Matrix;

所有线性层统一约定：y = x @ WT，W 形状 (out, in)。

MultiHeadAttention

Wq, Wk, Wv, Wo   // 各 (d_model, d_model)
num_heads          // 注意力头数
head_dim           // = d_model / num_heads

四把钥匙：Wq="我要找什么"，Wk="我有什么特征"，Wv="我的内容"，Wo="缝合多头结果"。

FeedForward

W1   (d_model, d_ff)    // 升维
b1   (1, d_ff)
W2   (d_ff, d_model)    // 降维
b2   (1, d_model)
activation               // GELU/R etLU/SiLU

漏斗：先放大到 d_ff 维，激活，再缩回 d_model 维。

3.3 Softmax 与 LayerNorm

Softmax（数值稳定版）

反向：dx[i] = y[i] * (dy[i] - Σ dy[j]*y[j])

直觉：每个输入的梯度不光看"自己那份"，还要减去"全局加权平均"。

解码器层比编码器层多什么？

  • 多一组 Cross-Attention：Q 来自解码器，K/V 来自编码器输出
  • 多一组 LN：ln3_gamma / ln3_beta
  • 自注意力带因果掩码：只能看当前位置及之前的位置

Pre-LN（本项目使用）

  • LN 在子模块 之前
  • 残差连接的梯度路径更短
  • 训练更稳定，不需要 warmup
  • GPT-2、LLaMA 等现代模型的选择

Post-LN（原始论文）

  • LN 在残差 之后
  • 梯度要穿过 LN 才能走捷径
  • 训练早期梯度不稳定
  • 需要 learning rate warmup

梯度流示意：Pre-LN 的残差路径（绿色虚线）没有 LN 阻碍，梯度直接传递。

Embedding 梯度是散点加法

Embedding 的梯度不像权重矩阵那样是矩阵乘法——只有 被查表查到的行 才有梯度：

for each (b, s):
    grad_embedding[src_ids[b][s]] += grad_encoder_input[b][s]

这就像一本字典：只有翻到的页才有修改记录，其余页原封不动。

问题：每生成一个 token 都要重跑完整 Encoder + Decoder。总计算量 = O(1 + 2 + ... + N) = O(N²)。

关键：编码器只跑一次，cross-attention 的 K/V 预投影后缓存。每步只跑一个新 token 的 decoder step（O(1)），不再重算历史。总计算量 = O(N)。
